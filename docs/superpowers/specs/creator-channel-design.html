2. AS-IS → TO-BE

AS-IS (현재)

  • Admin만 콘텐츠 업로드/관리
  • 콘텐츠에 소유자 개념 없음
  • User = 시청자 역할만
  • 단일 카탈로그 (넷플릭스형)
  • 채널/크리에이터 시스템 없음

TO-BE (변경 후)

  • 모든 User가 업로드 가능 (오픈 UGC)
  • 콘텐츠 → 채널 소속 (channel_id FK)
  • User → 채널 소유자 = 크리에이터
  • 카탈로그(넷플릭스) + 채널 페이지(유튜브) 공존
  • 채널 구독(팔로우) 시스템
  • 기존 콘텐츠 → 공식 채널 마이그레이션

4. Dual View System — Netflix + YouTube

Netflix View (홈/카탈로그)
/api/app/feed — 큐레이션 피드
/api/app/catalog/browse — 카테고리별 탐색
/api/app/catalog/search — 통합 검색

채널 무관, 모든 PUBLISHED 콘텐츠 대상
운영진 피처드 + 알고리즘 추천
기존 API 그대로 유지 (하위호환)
YouTube View (채널 페이지)
/api/app/channels/{handle} — 채널 프로필
/api/app/channels/{handle}/contents — 채널 콘텐츠
/api/app/channels/{handle}/series — 채널 시리즈

채널 소속 콘텐츠만 표시
크리에이터 브랜딩 (배너, 프로필)

5. New API Endpoints

Creator APIs — /api/app/creator (JWT 필수)

GET/api/app/creator/channel내 채널 정보 조회
PUT/api/app/creator/channel채널 정보 수정 (이름, 설명, 이미지)
POST/api/app/creator/contents콘텐츠 업로드 (multipart)
GET/api/app/creator/contents내 콘텐츠 목록
PUT/api/app/creator/contents/{id}/metadata메타데이터 수정
PATCH/api/app/creator/contents/{id}/status공개/비공개/삭제

GET/api/app/creator/series내 시리즈 목록
PUT/api/app/creator/series/{id}시리즈 수정

Channel Public APIs — /api/app/channels (Public)

GET/api/app/channels/{handle}채널 페이지 조회
GET/api/app/channels/{handle}/contents채널 콘텐츠 목록 (페이징)

DELETE/api/app/channels/{handle}/subscribe구독 해제 (JWT)
GET/api/app/me/subscriptions내 구독 채널 목록 (JWT)

Admin Channel APIs — /api/admin/channels (Admin JWT)

GET/api/admin/channels전체 채널 목록

7. DB Migration Strategy (V16)

Step 1. channel, channel_i18n, channel_subscription 테이블 생성
Step 2. content 테이블에 channel_id 컬럼 추가 (nullable)
Step 3. series 테이블에 channel_id 컬럼 추가 (nullable)
Step 4. "AI OTT Official" 시스템 채널 생성 (is_official=true, owner_id=NULL)
Step 5. 기존 content/series의 channel_id → 공식 채널 ID로 UPDATE
Step 6. channel_id에 NOT NULL 제약조건 추가
Step 7. 인덱스 생성 (channel.handle, channel.owner_id, content.channel_id, series.channel_id)
